## ⚠️ 参考サイト準拠ルール（最重要・厳守）

### 参考サイト
- **メインページ**: https://www.thomasmonavon.com/
- **詳細ページ**: https://www.thomasmonavon.com/gregory-lalle

### 厳守事項

⛔ **以下の要素は参考サイトと完全に同じにすること。独自の実装や逸脱は禁止。**

| 要素 | ルール |
|------|--------|
| 画面遷移アニメーション | 参考サイトと同一のタイミング・イージング・動作 |
| 文字のフォント | 参考サイトと同一（またはできる限り近いもの） |
| 文字の表示アニメーション | 参考サイトと同一（左下から広がる等） |
| スクロール挙動 | 参考サイトと同一の速度・イージング |
| カーソル表示 | 参考サイトと同一（Open/Close表示） |
| 詳細ビュー遷移 | 参考サイトと同一のアニメーション |

### 作業前の確認事項

1. **参考サイトの該当部分を必ず確認してから実装する**
2. **独自のアニメーションや挙動を追加しない**
3. **「これでいいだろう」という推測で実装しない**
4. **不明な場合はユーザーに確認する**

### 進捗管理ファイル
- **problem.txt**: 問題一覧
- **progress.txt**: タスク進捗管理

## ⚠️ 修正ループルール（厳守）

### 修正フロー

```
修正 → チェック → (問題あり) → 修正 → チェック → ... → (問題なし) → ユーザー確認待ち
```

### 各フェーズの説明

1. **修正フェーズ**
   - 参考サイトを分析
   - コードを実装
   - デプロイ

2. **チェックフェーズ**
   - 問題点が解決できているか確認
   - 参考サイトと同様の実装ができているか確認
   - 問題があれば修正フェーズに戻る

3. **ユーザー確認フェーズ**
   - 修正ループをパスした後、ユーザーに確認を依頼
   - ユーザーが問題なしと確認して初めて「解決」とする

### 厳守事項

⛔ **以下は絶対禁止**

| 禁止事項 | 理由 |
|----------|------|
| ユーザー確認なしに次のタスクに進む | タスク完了はユーザーが判断する |
| チェックフェーズを省略する | 品質を担保するため必須 |
| 「おそらく大丈夫」で完了とする | 必ず確認すること |

## ⚠️ 変更禁止ルール（最重要・厳守）

### 絶対に変更してはいけない要素

⛔ **以下の要素は現在の実装を維持すること。変更・修正・削除は絶対禁止。**

| 要素 | 説明 |
|------|------|
| トップ画面の見出しパーティクルアニメーション | `ParticleText`コンポーネントの実装。幾何学的パーティクルが散らばった位置から集まってテキストを形成するアニメーション。 |

**理由**: これらの要素はユーザー承認済みの完成した実装であり、変更すると既存の動作が壊れる可能性がある。

## ⚠️ 実装後チェックルール（最重要・厳守）

### 実装後の必須チェック項目

**実装・修正後は必ず以下を確認すること：**

1. **指示された機能が正しく動作するか**
2. **指示された部分以外に変更が加わっていないか**
3. **既存の機能が壊れていないか**
   - スクロールアニメーション
   - パーティクルアニメーション
   - 詳細ビュー遷移
   - カーソル表示
   - その他全ての既存機能

### チェック→修正ループ

```
実装完了 → 動作チェック → 問題発見 → 修正 → 動作チェック → ... → 問題なし → ユーザーに報告
```

⛔ **チェックせずにユーザーに報告することは絶対禁止**

### 変更影響範囲の確認（必須）

⛔ **修正を行った際は、修正箇所以外に影響が出ていないか必ず確認すること**

コードを変更した場合、以下を確認：
- CSS変更時：他の要素のレイアウト・アニメーションに影響がないか
- JS変更時：他のコンポーネント・機能に影響がないか
- 構造変更時：既存のスタイルやイベントハンドラが正しく動作するか

**特に確認すべき機能：**
- ホーム画面のスクロール動作
- 詳細ビューの開閉アニメーション
- 詳細ビュー内のスクロール
- パーティクルアニメーション
- カーソル表示
- ホバー効果

**影響が出ている場合は、その修正も同時に行うこと。**

## ⚠️ コーディングルール（厳守）

### 色の表記ルール

⛔ **RGB/RGBA表記は一切使用禁止。必ず16進数表記を使用すること。**

| 禁止 | 使用すべき表記 |
|------|---------------|
| `rgb(211, 207, 204)` | `#D3CFCC` |
| `rgba(211, 207, 204, 0.7)` | `#D3CFCCB3` |
| `rgba(0, 0, 0, 0.5)` | `#00000080` |

**透明度の変換表:**
| 透明度 | 16進数 |
|--------|--------|
| 1.0 | FF |
| 0.9 | E6 |
| 0.8 | CC |
| 0.7 | B3 |
| 0.5 | 80 |
| 0.3 | 4D |
| 0.2 | 33 |
| 0.15 | 26 |
| 0.1 | 1A |

## Tech Stack
- **Framework**: React (SPA - Single Page Application)
- **Styling**: CSS (vanilla)
- **Hosting**: GitHub Pages

## Architecture Rules

### React SPA
- シングルページアプリケーション形式で構築
- React 18を使用
- クライアントサイドルーティング（必要に応じてReact Router）
- コンポーネントベースの設計

### GitHub Pages Deployment
- `gh-pages`ブランチにデプロイ
- ビルド成果物は`/docs`または`/build`ディレクトリに出力
- 相対パスを使用（`homepage`設定に注意）
- SPAのため、404.htmlによるリダイレクト対応が必要
